Brush

Brush简介

一个Less库，提供了大量的易用性短标签和方法，用于快速搭建可用网站前端界面和UI。例如本页面是我用一天时间搭建起来的（呵呵，怎么可能）

出于对简化开发的探索，Brush尝试在Less的基础上，结合CSS3进行大规模的精简、组合。我们不满足于基础样式的样式标签简化，而在通用性装饰和动画上作出进一步探索，尝试将样式代码封装成一个个方法以供调用。就像我们曾经在C语言或其他编译型语言中所做过的事一样。

复合样式、组件

复合样式、组件会将基础样式组合起来成为一个可以通用的组件，自由添加在需要的节点上。同样为了保证定制化，我们将其做成方法以供调用，而且大部分组件会提供一个或几个默认样式用于快速添加。由于复合样式、组件是由多个基本样式组成，其样式难免会被既有样式覆盖，导致样式错误，故而请谨慎使用；避免此问题的最佳途径就是页面结构部件化（Componentization），降低样式耦合性。

为保证样式与结构分离，我们会尽最大努力不去添加任何可能的多余文档标签，例如装饰用的<em>或者<span>，而是使用被装饰的标签自带的伪类例如:before和:after。

Brush原则

秉持微小原则

保持任何可用标签、方法的简单，易用。为绝大部分方法提供默认参数，使其可以当成一个标签来使用。几乎所有标签都可以自由组合，当然部分互斥的样式标签会遵循CSS层叠原则，也就是前者被后者覆盖；而占用:before和/或:after的样式和方法十分建议单独使用。

谨慎添加

我们相信每个使用者和他背后的设计师都极具创造力，故而我们仅提供最基础，最简单的样式以及动画，以供自由组合。

当Brush的使用者希望添加一个导航栏时，Brush并不会提供一个.nav( ... )来供其调用（当然未来某个UI库可能会如此），而是在自己的项目中自行添加一个.nav { ... }来进行编写和细化，一个固定的方法调用可能会省却很多时间，但是却极大遏制了设计的完整性。

比如本页面左边导航栏下方的装饰线，只需要寥寥几行代码即可添加完成，甚至打包作为一个控件的通用部分，可Brush中并没有添加它（将来可能会添加一个相似但更通用的装饰线），对添加任何样式和动画保持谨慎，是Brush 的原则之一。

通用原则

我们希望添加的内容具有极大的通用性，故而在添加的过程中Brush方法会暴露大量的参数结构，甚至将某些可能需要js参与的样式剔除出Brush，毕竟在运行时添加less代码并不那么容易。

例如，.arrow()三角装饰不仅可以用在下拉菜单和提示框的上面，而且可以放置在类微信对话框中，人物对话框和头像之间；而.shadow()阴影更是可以放在几乎任何节点的下面作为装饰

小白勿近原则

了解层叠，了解flex，了解动画，了解一些css的深坑是小白应该做的事情，而Brush会屏蔽所有细节，以至于当出现问题时无法正确排除导致连锁反应。当然这里说的有些绝对，本意是指在使用Brush之前，你需要一些基础知识作为支持。
